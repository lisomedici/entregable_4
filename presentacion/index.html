Del modelo de churn a una herramienta operativa para Kavak Seguros

API local, dashboard de priorizacion y demo en vivo para contactar polizas con mayor riesgo de baja temprana.

Problema

Kavak necesita actuar antes de que la poliza se cancele.

25,5%de polizas con churn temprano en el set modelado.
M0-M3ventana critica definida para detectar baja temprana.
Reactivosin score, retencion llega tarde o prioriza a ciegas.

Modelo anterior

El Entregable 3 dejo una senal predictiva validada out-of-time.

XGBoostmodelo ganador contra Logistic Regression y Random Forest.
49,1%Precision@Top20%, por encima del KPI objetivo de 40%.
0,731ROC-AUC en test out-of-time.

Implementacion

Operacionalizamos el caso de uso con servicio + interfaz.

1. CSVLa API lee el listado de polizas y calcula features seguras.
2. ScoringEl servicio devuelve score, nivel de riesgo y drivers.
3. DashboardLas asesoras ven el ranking accionable de contactos.
4. DemoPrediccion individual y lista priorizada en vivo.

Salida operacional

El score se traduce en una accion concreta.

AltoScore mayor a 0,40. Contacto proactivo en la primera semana.
MedioScore entre 0,20 y 0,40. Seguimiento preventivo durante onboarding.
BajoScore menor a 0,20. Flujo normal sin intervencion especial.

Pasamos de un notebook a una herramienta usable.

ValorFoco en las polizas donde una llamada temprana tiene mayor probabilidad de evitar churn.
